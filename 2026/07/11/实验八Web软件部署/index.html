实验八 软件部署

1、实验目的

本次实验旨在综合运用前七个实验所学的各类工具，构建一个完整的网络环境，完整体验Web静态站点从开发到部署的全过程。通过GitHub Actions实现自动化部署（CI/CD），并掌握将静态站点部署到云服务器（Nginx）的方法，深入理解持续集成与持续部署的核心概念与实践流程。

  • 虚拟化平台：Oracle VirtualBox 7.0
  • 虚拟机配置：内存2GB，硬盘20GB
  • Node.js版本：v20.11.0
  • npm版本：v10.2.4
  • Git版本：v2.42.0
  • Hexo版本：v7.0.0
  • Nginx版本：v1.18.0
  • 部署平台：GitHub Pages、Ubuntu Server（Nginx）
  • 博客框架：Hexo（Butterfly主题）
  • CI/CD工具：GitHub Actions

5、知识总结

5.1 CI/CD核心概念

概念 含义 本实验中的实现
持续集成（CI） 代码变更后自动进行构建和测试 推送代码到main分支后自动执行hexo generate
持续部署（CD） 构建通过后自动部署到生产环境 构建完成后自动推送到GitHub Pages

5.2 静态站点部署方式对比

部署方式 适用场景 优点 缺点
GitHub Pages 个人博客、开源项目文档 免费、支持HTTPS、与Git原生集成 不适合动态内容
Nginx（虚拟机/云服务器） 企业应用、需完全控制的环境 完全可控、支持自定义配置 需自行维护服务器

5.3 常用Nginx命令

命令 作用
sudo systemctl start nginx 启动Nginx服务
sudo systemctl stop nginx 停止Nginx服务
sudo systemctl reload nginx 重载配置（不停机）
sudo nginx -t 测试配置文件语法
sudo ln -s sites-available/blog sites-enabled/ 启用虚拟主机

7、心得体会

技术层面：

通过本次实验，我完整地体验了一个Web应用从开发到部署的全生命周期。从实验二的Node.js环境搭建、实验三的代码编辑器配置、实验四的Hexo博客构建，到实验五至实验七的虚拟化与远程管理，再到本次实验的自动化部署，整个过程让我深刻理解了现代软件开发中”DevOps”理念的重要性。

GitHub Actions作为CI/CD工具，让我第一次感受到了自动化部署的强大魅力——只需一次git push，代码就能自动构建并部署到全球可访问的网站上。这种”提交即上线”的工作流极大地提高了开发效率。

rsync工具的增量同步功能也让我印象深刻，它只传输变化的部分，非常适合频繁更新的静态站点部署场景。

思政层面：

本次实验使用的所有工具——Hexo、Node.js、Git、Nginx、Ubuntu、VirtualBox——全部是开源软件。这让我深刻体会到了开源社区的强大力量和共享精神。这些优秀开源项目汇聚了全球开发者的智慧，让我们能够以零成本完成一个完整的Web应用部署流程。

同时，GitHub Actions的自动化部署能力也让我思考：我国虽然有Gitee、Coding等代码托管平台，但在CI/CD生态建设和社区活跃度方面仍有提升空间。作为未来的软件工程师，我不仅要学会使用这些优秀的开源工具，更应当积极参与到开源社区中，学习开源项目的设计思想和协作模式，为国产软件生态的发展贡献力量。

此外，在配置Nginx和rsync的过程中，我深刻认识到在生产环境中每一个细节都至关重要——配置文件的一处错误可能导致服务不可用，权限设置不当可能带来安全隐患。这种严谨的技术责任感是每个软件工程师必备的素养。
