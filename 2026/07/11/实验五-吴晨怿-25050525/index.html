一、实验目的

  1. 使用虚拟机工具（VirtualBox）安装Linux系统（Ubuntu Server）；
  2. 通过克隆技术构建一个多机虚拟化环境；
  3. 了解和掌握基于虚拟机的开发环境的基本构建方法和配置。

二、实验环境
1.操作系统：Windows 10/11
2.网络：需要互联网连接
3.硬件要求：CPU支持VT-x/AMD-V虚拟化技术；内存≥8GB；硬盘剩余空间≥50GB
4.软件：Oracle VirtualBox Ubuntu Server 22.04 LTS ISO镜像

三、实验过程
1.下载安装VirtualBox和Ubuntu Server ISO 镜像，根据实验手册提示配置
2.创建虚拟机，根据实验手册配置相关环境

五、知识总结

1、核心流程

VirtualBox安装 → 创建模板机（Ubuntu Server，双网卡） → 配置静态IP与主机名 → 克隆生成node1/node2/node3 → 修改各克隆机IP与主机名 → 网络连通验证

2、核心知识点

知识点 关键内容 命令 / 配置
VirtualBox 网络模式 NAT（enp0s3）：上网用，DHCP自动获取IP；
仅主机（enp0s8）：虚拟机间通信，手动设静态IP		 虚拟机设置 → 网络 → 适配器1（NAT）、适配器2（仅主机）
Netplan 静态IP 修改 /etc/netplan/50-cloud-init.yaml，YAML语法（只用空格缩进） sudo nano /etc/netplan/50-cloud-init.yaml
sudo netplan apply
仅主机网卡配置 固定IP，所有虚拟机必须在 192.168.56.0/24 网段 dhcp4: no
addresses: [192.168.56.x/24]
主机名与hosts 每台机器设置唯一主机名；/etc/hosts 统一配置IP‑主机名映射 sudo hostnamectl set-hostname <名字>
sudo nano /etc/hosts
关闭cloud-init 防止重启后静态IP配置被自动覆盖 创建 /etc/cloud/cloud.cfg.d/99-disable-network-config.cfg
写入 network: {config: disabled}
克隆注意事项 必须选择“完全克隆”并勾选“重新生成MAC地址”，否则网络冲突 VirtualBox 右键 → 克隆
连通性测试 验证外网、宿主机、虚拟机间互通 ping -c 3 8.8.8.8（外网）
ping -c 3 192.168.56.1（宿主机）
ping -c 3 node2（虚拟机间）

3、集群IP规划

主机名 静态IP（仅主机网卡）
template 192.168.56.101
node1 192.168.56.102
node2 192.168.56.103
node3 192.168.56.104

六、出现问题
在创建虚拟机后，双击无法启动，提示-104错误。根据B站相关视频下评论指引，发现是电脑中游戏平台一些反作弊系统阻挠虚拟机正常运行，在卸载相关游戏平台后，虚拟机得以正常启动。
