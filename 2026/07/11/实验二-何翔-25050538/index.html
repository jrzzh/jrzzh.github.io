1. 实验目的

本次实验旨在安装并配置 Visual Studio Code 编辑器，掌握 Markdown 语法及实时渲染预览方法；同时学习 Git 版本管理工具的基本操作，包括本地仓库管理、分支操作及与远程仓库（GitHub）的同步，为后续软件开发中的文档编写和代码版本控制打下基础。

步骤五：分支操作（创建 dev 分支并合并）

5. 知识总结

Git 核心命令

命令 功能说明 本次实验使用场景
git --version 查看 Git 版本号 验证 Git 是否安装成功
git config --global user.name "..." 设置全局用户名 配置提交身份
git config --global user.email "..." 设置全局邮箱 配置提交身份
git config --global --list 查看所有全局配置 验证身份配置是否生效
git init 在当前目录初始化 Git 仓库 创建本地仓库
git add <file> 将文件添加到暂存区 准备提交文件
git add . 添加当前目录所有变更到暂存区 批量添加文件
git commit -m "message" 提交暂存区内容并附上说明 保存版本快照
git log --oneline 查看简洁的提交历史 查看提交记录
git log --oneline --graph 查看图形化的提交历史 查看分支合并关系
git checkout -b <branch> 创建并切换到新分支 一步创建+切换 dev
git merge <branch> 将指定分支合并到当前分支 将 dev 合并到 master
git remote add origin <url> 关联远程仓库 连接 GitHub 仓库
git push -u origin master 推送本地分支到远程并设置上游 首次推送本地代码到 GitHub
git status 查看当前工作区状态 检查文件是否被跟踪

6. 出现问题及解决方案

6. 出现问题及解决方案

问题描述 原因分析 解决方案
终端中按退格键无法删除字符 开启了中文输入法 切换为英文输入法（美式键盘）后再操作
git commit 时提示 “please tell me who you are” 未配置 user.name 和 user.email 执行 git config --global user.name 和 git config --global user.email 设置身份信息
在桌面直接执行 git init 误操作导致桌面被 Git 跟踪 使用 rmdir /s .git 删除桌面上的 .git 文件夹，新建专用文件夹重新初始化
git push 时提示认证失败 GitHub 已禁用密码登录，需使用 Token 在 GitHub Settings → Developer settings → Personal access tokens 生成 Token，作为密码输入
终端路径不显示当前所在文件夹 未进入目标目录 使用 cd 文件夹名 进入正确目录，用 pwd 验证当前位置
VS Code 终端显示 >> 无法执行命令 上一行命令未结束（如漏了引号或括号） 按 Ctrl+C 终止当前输入状态，重新输入正确的命令
